1.How to debug CSS

  • console command to show errors
  • elements to inspect the part that isn't looking how you want it to
  • order of priority = inline, internal, then external
  • best practice is to have all styles in external css

css syntac

  • selector {prperty:value;)
  • best practice is to have all styles in external css
  • best practice is to have properties in alphabetical order

types of selectors

  • tag selector
    • weakest, affects anything with that tag
  • class selector
    • stronger than tag selector, used when you want to only change a 1->a few elements in the same way
    • its posible to use two classes on the same element
  • id selector
    • strongest, can only be used for one element on page. never repeated
    • try not to use a lot

css display property

  • display:inline
    • Displays an element as an inline element (like span). Any height and width properties will have no effect
  • display:block
    • Displays an element as a block element (like p). It starts on a new line, and takes up the whole width
  • display:inline-block
    • Displays an element as an inline-level block container. The element itself is formatted as an inline element, but you can apply height and width values
  • display:none
    • The element is completely removed
    • rather than using this use visability: hidden to make elements act as placeholders

css position property

  • static
    • go with the normal html rules. (default for any html without css)
  • relative
    • posiion the element that we select releative to how it would have been. (with margins)
      1. top
      2. bottom
      3. left
      4. right
    • ! when you move an element the other elements still pretend you didnt move anything
  • absolute
    • postion element relative to it's parent
    • does affect the flow of the elements
  • fixed
    • fixed to position of website. if you scroll it always shws
    • this would be good for a nav bar or something that always should go with the user as they scroll
